# n8nで魂をJSON化したパイプラインをClaude Codeで再設計したら、同じ問題を違うレイヤーで解いていたと気付いた話

## はじめに
思考の拡張、いわゆる「自己拡張システム」を作ろうと試行錯誤していた時期がある。
最初は **n8n** を使っていた。自分の思考プロセスや日々の記録（いわゆる外部脳への書き出し）を、ノードとエッジで繋いで自動化していく。
それは文字通り、**「自分の魂をJSON形式に構造化して抽出する」** という途方もない好奇心（未知への探求心）から始まった作業だった。視覚的に思考のフローが組み上がり、美しいパイプラインが出来上がっていった。

## Claude Code（CLI）への移行
ある時、このシステム全体を **Claude Code** (CLI環境) で再設計してみようと思い立った。
ブラウザ上の綺麗なGUIを手放し、ターミナルの黒い画面とプロンプトだけで、自分の魂のJSONを解釈できるAIエージェントを構築するアプローチへの変更だ。

最初は「まったく新しいシステムを作り直す」つもりでいた。しかし、コードを書き、CLIでClaudeと対話しながらシステムを再構築していくうちに、ある強烈な既視感に襲われた。

## 結局、同じ問題を違うレイヤーで解いているだけだった

「あれ、これn8nのWebhookとSwitchノードで作ってた分岐と全く同じことを、プロンプトのコンテキスト管理としてやってるだけじゃないか？」

そう。私はツールを変えただけで、直面している本質的な課題は**何も変わっていなかった**のだ。
n8nでは「データの受け渡しパスの設計（JSONの構造）」というレイヤーで課題を解いていた。
一方のClaude Codeでは、「LLMにどう文脈を保たせるか、CLIからどのツールを呼び出すか」というプロンプトとインフラのレイヤーで同じ課題を解いていた。

抽象度が「視覚的パイプライン」から「コマンドライン・プロンプト」へと上下に移動しただけで、**「自分の思考をどうシステムに委譲するか」という根本的な魂（コア）の設計論は全く同じ** だったのだ。

## 初心を忘れないことの重要性
この気付きは、エンジニアリングにおける大きな学びとなった。
私たちは往々にして「新しいツール（今回はClaude Code）を使えば、今までの問題がすべて消え去る」と錯覚してしまう。
しかし、どんなに技術が進化しても、解決すべき業務のコアパラメーター（目的）は変わらない。

「未知への探求心」を持って新しいレイヤー（CLI）に挑戦したからこそ、「あぁ、n8nでやっていたあの泥臭いJSONの構造化は、決して無駄じゃなかったんだ」と本質を再確認できた。
どんなツールを使おうとも、常に「自分が今、どのレイヤーの何を解決しようとしているのか」を問い続けること。この**「初心を忘れない態度」**こそが、一生使い続けられる本当の自己拡張システム（統合思考OS）の土台なのだと思う。
